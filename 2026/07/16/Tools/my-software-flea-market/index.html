来逛逛我的软件杂货铺：这些年我用过的工具

Henry

发布于：2026年7月16日

次浏览

字数：7.7k 字

时长：26 分钟

电脑里总有一些软件，平时几乎感觉不到它们的存在；直到换了一台新电脑、需要处理一批文件，或者某个操作突然变得很麻烦，才会发现它们早已嵌进了自己的习惯里。

这篇文章就是我的软件杂货铺。里面有每天开机就出现的常驻工具，也有只在某个具体场景才会打开的“专用扳手”；还有一些是以前常用、现在很少动的旧工具。它们未必适合所有人，也不构成一份“Windows 必装软件清单”。把它当成逛跳蚤市场可能更合适：大多数东西你未必需要，但说不定正好能淘到一件自己之前不知道该去哪里找的东西。

我的主力系统是 Windows，所以下面的具体软件也大多来自 Windows 生态。Mac 用户不必因为软件名称不同就划走：更重要的是先发现一个需求原来可以被工具解决，再去找对应平台上更合适的替代品。很多效率提升，开始于“原来还能这样做”。

文中的价格、套餐和维护状态都可能变化，我会尽量说明成本类型和自己的实际使用方式；涉及付费软件，优先建议从官方渠道或开源项目获取。网上确实能找到一些软件资源导航站，例如克隆窝和绿软小站，但第三方资源的版权、版本完整性和安全性都需要自己判断，尤其不要拿来处理敏感资料。

先从每天都在减少绕路的工具说起

这一组工具的共同点是：它们不替你完成某项宏大的工作，只是持续减少电脑使用中的小摩擦。单次节省几秒钟不显眼，日积月累下来很可观。

PowerToys：把常用动作叫到手边

PowerToys 是微软官方的 Windows 效率工具集，免费且开源。它的功能很多，但我最常用的其实只有命令面板：按 Alt + Space 呼出后，输入应用名称，就可以快速打开软件，也可以执行关机、重启这类系统操作。

这听上去并不惊天动地，但它把“找应用”从鼠标操作变成了输入一个词。对我来说，这意味着不必在桌面、开始菜单和任务栏之间找来找去；知道自己要打开什么，打出几个字就够了。用久之后，这种入口会慢慢变成肌肉记忆。Mac 上可以把这类需求理解为 Raycast 或 Spotlight 一类的启动器需求，不必执着于一比一的产品对应。

这也让我慢慢理解了为什么很多熟练用户会偏爱键盘操作。无论是程序员熟悉 Vim 后尽量不离开键盘，还是金融从业者用快捷键搭 Excel 模型，追求的都不是“按键很酷”，而是让已经想清楚的动作直接落地。鼠标需要移动、定位、确认；键盘则把常用操作编码成一组离散的输入。熟练之后，手不必在键盘和鼠标之间来回，注意力也少一次在界面里寻找按钮的过程。

PowerToys 还有一个我最近重新注意到的功能：Light Switch。按 Shift + Ctrl + Win + D 可以手动切换明暗模式。我以前不怎么用，最近打算把它纳入习惯。这里也正好说明了工具的另一面：装着不等于真正进入工作流，真正有用的是那些能在恰当时机被想起来的功能。

Everything：记得文件名，就不用记得它住在哪儿

Everything 是经典的 Windows 文件名搜索工具。我大多数时候只是直接搜文件名，很少使用高级检索语法；这已经解决了最常见的问题：我记得那份文件大概叫什么，却不记得它在哪个目录、哪一层文件夹里。

这类情况很常见：某份报告、一个下载过的安装包、一张以前保存的图片，名字还留在脑子里，路径却早就消失了。Everything 把“回忆文件夹结构”变成“输入几个关键词”。

Windows 自带搜索这些年在进步，但 Everything 的体验仍然很干脆。它更像一个文件索引入口，而不是另一个复杂的文件管理器。高级检索在需要时也能派上用场，不过我不建议把学习语法当作使用门槛，先拿它替代“手动翻文件夹”就够了。

TrafficMonitor：电脑的仪表盘

TrafficMonitor 是我开机自启动、常驻任务栏的工具。它显示 CPU、内存占用和上下行网速，我几乎一直让它在那里。

我会把它比作汽车仪表盘。平时只是余光扫一眼；当电脑突然卡顿、下载速度异常、某个程序像是在偷偷吃内存时，仪表盘会先告诉你“哪里不太对”。做软件开发、跑数据分析任务时尤其方便，但它并不只属于技术场景。至少在我这里，它让排查问题从“感觉电脑有点慢”变成“CPU、内存还是网络出了问题”。

有时它甚至不需要帮我解决问题。知道电脑此刻运行得很正常，本身也能省掉一点无谓的猜测。

酷呆桌面：让桌面在不同显示器之间复原

酷呆桌面（Coodesker） 是付费买断的桌面管理工具，也是我的开机自启动软件。它可以在桌面上建立盒子、把文件夹映射进盒子，桌面看起来会整洁很多。每天都要面对的桌面，顺眼一点总不是坏事。

它真正解决我的问题，是显示器切换。我在工位会接外接显示器，外出时只能带笔记本，两个屏幕的桌面布局很难共用。切回笔记本时，原来放在外接屏上的盒子容易失去位置；回到工位，又要重新把常用内容摆开。

我的做法是先分别把两种模式手动调整到满意的状态：一份是笔记本单屏布局，一份是外接显示器布局，然后各自导出布局备份文件。之后切换环境时，直接导入对应模式的备份文件即可。

这套东西也需要维护。某个模式下调整了盒子的位置、映射内容或桌面分区，满意后我会立刻再导出一份新的备份，覆盖原来的版本；如果想更稳一点，也可以只保留最近几次备份。软件自身还有自动备份功能作为兜底。

听起来像是在认真管理桌面布局，确实如此。但对经常在两种屏幕环境之间往返的人来说，这比每次重新收拾桌面轻松得多。

XYplorer、Geek Uninstaller 和 Quicker：不必天天开，但需要时很省事

XYplorer 是一个付费文件管理器。我现在并不拿它替代 Windows 文件资源管理器，主要只在清理磁盘时打开：它可以一键统计当前目录下各个文件和文件夹的占用空间，顺着体积往下找，很容易定位“硬盘到底被谁吃掉了”。

Geek Uninstaller 则是我卸载软件时的默认选择。卸载本身不稀奇，关键是它会扫描并清理卸载残留。电脑用久了以后，很多软件留下的配置、缓存、空目录并不显眼；我不追求把系统打扫成无菌室，但能顺手清掉就清掉。它提供免费基础版，也有 Pro 版本，具体功能和价格以官网为准。

Quicker 曾经是我用得很多的效率工具。截图 OCR、窗口置顶、学术场景下的公式识别都很实用；它的剪贴板、云板、图片压缩、批量重命名、窗口调整等动作也可以通过插件补齐。

现在我已经很少用它了。尤其是批量重命名这类需求，经常会交给 Codex 之类的 AI 工具来处理；但截图 OCR、窗口置顶和公式识别仍然说明，很多看似零碎的需求可以被做成随手可调用的小动作。AI 没有让所有旧工具失效，但确实重新分配了它们在工作流里的位置。

文档、PDF 和“把资料喂给 AI”

我日常接触的资料里，PDF、Markdown 和 Office 文档占了很大一部分。这里的工具不是要把所有文件统一到一个软件里，而是把“阅读”“编辑”“批量处理”“结构化提取”分给更适合的工具。

Typora：我最常打开的 Markdown 阅读器

Typora 是我使用频率很高的付费 Markdown 编辑器。现在我经常阅读 AI 生成的项目文档，Typora 的所见即所得、目录和导出功能让这件事舒服很多。它把 Markdown 从“给工具看的纯文本”变成了更适合人阅读的文档界面，既不会让 Markdown 满屏都是标记，也没有把文件锁进某个专有数据库。

主题也会直接影响这种阅读体验。一套合适的主题能让标题层级、表格、引用、代码块和导出 PDF 都更舒服，长文读起来也更愿意停下来。我自己做了一套 Lumen Typora Theme，提供 Azure 与 Forest 两组配色、明暗模式，并专门处理了 PDF 导出；感兴趣可以自行试试。

我之前专门写过一篇Typora、Markdown 阅读与 AI 时代工具改造的文章，这里不展开。简单说，如果你经常面对长 Markdown 文档，又希望它看起来像文档而不是源码，Typora 很值得试用。它是买断制付费软件，价格以官网为准。

MinerU：把复杂 PDF 变成可用的 Markdown

MinerU 是我觉得非常有 AI 时代特征的工具。它的目标不是简单“从 PDF 复制文字”，而是把研报、通知、资料、书籍等复杂文档转换为更适合大模型使用的 Markdown 或 JSON。项目的开源实现可以看 opendatalab/MinerU。

PDF 里真正麻烦的部分，往往不是“有没有文字”，而是阅读顺序、表格、公式、图片、扫描件 OCR、页眉页脚和双栏排版。很多 PDF 很“脏”，用普通库提取出来的文本几乎不能看，更谈不上直接作为 AI 的上下文。PDF 越复杂、越接近扫描件，MinerU 相比简单提取的价值就越明显。

对人来说，很多 PDF 的可读性其实高于 Markdown。它们经过排版，有明确的字体、图片、表格和页面结构；需要阅读原文时，我仍然更愿意打开 PDF，除非是在 Typora 这类阅读体验很好的 Markdown 工具里查看。

PDF 转 Markdown 的主要收益不在于替代原文，而在于给资料增加一层可处理性。高质量转换后，研报、通知、书籍和各类资料可以成为 AI 的优质上下文，交给 Codex 这类工具做检索、提问、整理、对照和后续处理。

这也是 MinerU 有价值的地方。普通文本提取常常会打乱阅读顺序，丢掉表格、公式或图片信息；MinerU 要做的是尽量保留这些结构，让转换后的 Markdown 真正可用。

我使用的是直接下载安装的桌面程序，应用不需要手动登录。按我目前账户看到的额度，每天可处理 5000 份文件，其中有 1000 页优先解析页数；超过优先页数之后仍可解析，只是可能需要排队。这个额度和策略未来都可能调整。

这里有一个比功能更重要的边界：本地桌面应用不必然意味着所有文件都在本地离线处理。涉及未公开项目、公司资料、个人隐私或其他敏感 PDF 时，应先确认产品当前的数据处理方式，再决定是否上传。这不是 MinerU 独有的问题，而是所有“把文件交给云端 AI 处理”的工具都绕不过去的基本判断。

PDF 的轻重搭配：SumatraPDF 和 Adobe Acrobat

SumatraPDF 是我的默认 PDF 阅读器。原因非常朴素：我最高频的动作是阅读 PDF，它足够轻、打开快、没有多余打扰。默认工具应该优先优化最常做的事情，而不是把最强大的工具设成唯一入口。

需要复杂操作时，我会手动用 Adobe Acrobat 打开，例如编辑、处理表单、组织页面或其他重度 PDF 操作。Acrobat 是更完整的专业工具，也相应是付费订阅产品。这种“轻量阅读器做默认，重型编辑器按需打开”的组合，比让一个全能软件承担所有日常动作更顺手。

Print Conductor 和 7-Zip：批量打印与压缩解压

Print Conductor 是我以前在工作中用过的批量打印工具。例如要一次打印一批 Word 文档时，它比一个个打开、一个个点打印省事得多。我现在已经很久没用它了，但“批量打印多种文件”这个需求一直存在；它是商业软件，价格不低，按需评估即可。

7-Zip 是我电脑里唯一安装的压缩和解压工具，解压、压缩都用它。它免费开源，轻便可靠，也没有必要为这个需求装一堆功能相近的软件。

把想法、资料和行动放在不同的位置

笔记工具最容易陷入“到底哪个最好”的争论。我的感受是，不同工具承担不同类型的信息，反而比强行找一个万能系统更稳定。

我现在的分工可以简单写成：flomo 主要保存“我已经想过什么”，思源笔记主要保存“我正在研究什么，以及以后可能需要查什么”；微信读书提供书籍和划线入口，滴答清单把笔记压到任务和日程旁边。它们之间有重叠，但各自的重心不同。

滴答清单：任务复杂起来，就让任务自己带着上下文

滴答清单 是我高频使用的个人时间管理和任务管理工具。我愿意为它按月付费，自己使用时大约是每月 15 元；价格和套餐当然可能调整。

它不只是一张待办列表。一个任务变复杂时，任务本身可以变成轻量项目页：行动、背景、沟通记录、下一步安排放在一起，任务完成后上下文也还留在那里。任务里的笔记支持 Markdown，这让它在某些时候也像一个和日程绑定的微笔记系统。

滴答清单里的笔记不是为了沉淀所有知识，而是服务于把一件事推进下去。资料库可以放在思源，随想可以进入 flomo；但当一个具体事项需要完成时，相关背景和下一步最好就待在任务旁边。信息沉淀在离现场最近的地方，被再次调用、查看和补充的成本也最低。

这也是我对“待办事项 App 是 vibe coding 新手三件套之一”的一点补充。很多人喜欢从零写一个待办应用，但对我来说，滴答清单已经把这个需求做得足够成熟。日程、提醒、重复任务、跨端同步和历史数据维护，往往比一个漂亮的勾选框难得多。未来我会把它和 flomo、思源笔记、微信读书放在一起，单独聊聊这套知识运营系统。

flomo：我的“思想大脑”

flomo 更适合承载我自己的输出。我尽量不把复制来的材料或 AI 生成内容直接堆进去，而是记录随想，或者把微信读书里划出的句子贴进来，再用自己的话写下当时的理解。

我不把这些想法只留在微信读书的划线旁边，是希望它们能和其他来源的念头放到同一个空间里。我的 flomo 标签大致分两类：一类标记来源，例如阅读、随想、日记、网页摘录；另一类标记主题，例如“主题/好奇”。于是同一个主题下，可以同时出现一次和 AI 对话后的思考、某篇公众号文章的笔记、一本书里的摘录和自己的回应。不同来源的内容有机会在这里交汇。

flomo 的多级标签和 AI 功能让这种整理不只停留在归档层面。它可以帮助发现笔记之间的关联，也给后续回看和重新组织留下空间。这也是我把阅读时的想法统一交给 flomo 管理的原因：它更像一个让观点彼此碰面的地方。

这种做法和 flomo 倡导的卡片笔记法很契合：一条卡片不宜太少，太少可以合并；也不宜无限长，太长就应该拆成更原子的想法。把内容重新说一遍并不只是“存档”，输出本身就是理解和整理的过程，有一点费曼学习法的意思。

flomo 做得好的地方还在于克制。即点即写、语音输入、标签自动补全、多端同步，这些功能不会抢走注意力。它对复杂输入保持了克制，但并非完全拒绝图片：flomo 支持上传图片，只是免费与付费计划的存储空间不同。相比公式、代码和复杂排版，它仍然更适合承载短小、以文字为中心的想法。最近它也推出了 AI 洞察、认知地图、语音转笔记、相关笔记等功能；我还没有系统体验，暂时只把它们看作值得继续观察的延伸。

我使用的是 flomo Pro，年费 99 元，购买时有学生优惠。价格政策可能变化，所以更重要的判断仍然是：如果你需要的不是资料仓库，而是一个低摩擦、鼓励主动输出的想法入口，它值得试一试。

思源笔记：我的知识生产工作台

思源笔记 是我知识运营系统里更 heavy 的部分。我更愿意把它看作一个“知识生产工作台”，而不只是资料库。

做数学题需要草稿纸，做研究同样需要一块能持续展开的空间：资料先落下来，再被整理、引用、补充和重组。目的不是打造一个永远完美的资料库，而是在不断收集、整理资料的过程中形成对主题的理解，并逐渐沉淀出可以交付、或者未来能再次调用的内容。

如果说 flomo 主要保存“我已经想过什么”，思源笔记更接近保存“我正在研究什么，以及以后可能需要查什么”。这里允许粘贴材料、保存 AI 生成内容，也允许一份资料比一张卡片长得多、结构复杂得多。项目资料、较完整的研究整理、需要多级文档和相互引用的内容，我会放进思源。

最近的 3.7.0 更新又让这个工作台多了一层可能：思源开始提供官方 CLI 和 MCP 的雏形。我专门为此做了一个让 Codex 协作使用思源的 skill。这样一来，思源不再只是我手动打开、手动整理的笔记软件；在合适的项目中，AI 可以协助检索和梳理资料、整理文档、起草报告，承担一部分重复、琐碎的 dirty work。

这并不意味着用户可以退出思考。问题该怎么定义、资料该信什么、项目往哪里推进，仍然需要人来判断。变化在于：当 AI 能进入知识生产工作台，用户可以少花一些时间做初级分析师式的信息搬运，把更多注意力放在把控方向、提出问题和做最终判断上。CLI 和 MCP 还在早期阶段，但这个方向很有潜力。

我曾经试过用 Typora 加本地文件夹管理笔记，也用过 Notion 和 Obsidian，最后还是选择了思源。它给我的感觉有点像“本地化的 Notion”：功能足够丰富，但本地数据、离线可用性和迁移成本更让人安心。对我而言，手机端主要用于查看，编辑还是放在电脑上完成。

它的付费方式也值得区分：软件本身免费；购买一次性功能特性后，可以接入自己的 S3 或对象存储进行同步；官方多端同步服务则是年付订阅。以我看到的当前页面为例，前者的促销价约为 80 元，后者年付约为 148 元，但活动、价格和对象存储本身的费用都可能变化。我的选择是配置自己的 S3 存储，让电脑上的更新能同步到手机上。

这三个工具放在一起，不是为了组成一个看上去很“高级”的系统，而是因为它们分别照顾了想法、资料与行动。工具越多，边界越重要；否则最后只会得到三四个同步混乱的收件箱。

屏幕与媒体：有的负责记录，有的负责不折腾

Bandicam：需要录屏时，我就打开它

Bandicam 不是我每天都会用的工具，但只要需要录制会议、操作过程或汇报内容，我通常就会打开它。它上手比较直观，功能也够完整；我不拿它录游戏，主要把它当作稳定的屏幕录制工具。

免费版通常会有时长或水印限制，正式使用需要购买许可。录会议时也别忘了更重要的一层：先确认录制是否获得参会者和组织方允许，以及文件该如何保管。

mpv、PotPlayer 和那些曾经处理过媒体文件的工具

mpv 是我现在的默认视频播放器。它免费开源、轻量，至少在我的电脑上感觉更不容易卡顿。这是一个很个人的体验，不意味着它一定比所有播放器都好；但如果你受够了臃肿播放器的广告、弹窗或拖慢感，值得了解一下。

PotPlayer 是我以前用得更多的经典 Windows 播放器，现在仍然保留着但很少打开。两者的区别大概可以理解为：PotPlayer 功能丰富、图形界面完整；mpv 更轻、更极简、更像“只管把视频放好”。

我曾经还用过 foobar2000 做音频播放、MusicTag 补全音频信息、小丸工具箱处理音视频、MediaCoder 压缩转码、SplitIt 切割视频。现在这些工具大多已经不在我的高频工作流里。把它们写出来不是为了推荐读者照单全收，而是保留一张需求地图：音频标签、转码压缩、无损切割，这些需求都可以被单独解决。真遇到时，优先找当前维护活跃、适合自己格式和平台的工具。

Wallpaper Engine：桌面也可以不只是空白背景

Wallpaper Engine 是 Steam 上的付费动态壁纸软件，我现在仍在使用，也设置成开机自启动。创意工坊里有大量壁纸可选，效果很好。

它不解决待办、文件或资料处理问题，就是让每天面对的桌面更合自己的心意。喜欢动态桌面的人可以试试；觉得静态壁纸已经很好，也完全没有必要为了“效率”给自己增加一个软件。

代码与 AI：留在清单里，但不在这里班门弄斧

我的代码相关工具主要有 Git、VS Code、Sublime Text 和 Codex App。VS Code 是主力编辑器，Sublime Text 适合轻量、快速地打开或编辑文本；Git 现在更多交给 Codex 操作，或者在 VS Code 的图形界面里查看变化，我已经很少手动敲 Git 命令。

Codex App 则是我高频使用的 AI 协作入口。它的价值不只在于“生成代码”，还在于能进入真实项目、读文件、改文件、运行命令、处理批量任务。于是某些以前必须依赖专门效率插件才能完成的事情，现在可以直接用自然语言交给它。

不过这一部分没必要写得太细：真有这些需求的读者，通常已经有自己的工具偏好和工作流。我把它们放进清单，只是因为它们确实在我的电脑里占了一席之地。

还有一个网络工具：Clash Verge

我目前使用的是 Clash Verge。它属于网络代理客户端，这里只提一句，不展开配置或使用方法。以前我用过 Clash，后来原项目出现维护中断等变化；现在常见的延续项目是 Clash Verge Rev。这也让我更在意工具的维护状态：再顺手的软件，如果长期没人维护，迟早会影响使用体验。涉及网络、账号与隐私的工具，尤其应该从可信来源获取，并确认当地法律、服务条款和组织规则。

工具会把一些原本看不见的需求照亮

我越来越觉得，工具的价值不只是把某个动作做快一点。很多摩擦在没有替代方案之前，甚至不会被意识到。

没有文件搜索工具时，人会把“翻半天文件夹”当成电脑使用的一部分；没有好的 Markdown 阅读器时，人也许默认接受“项目文档就是一堆源码”；没有 PDF 解析工具时，很多资料看起来天然不能进入 AI 工作流。工具出现后，我们才开始给这些体验命名，才发现原来还有另一种可能。

这让我想起麦克卢汉那句常被引用的话：我们塑造工具，尔后工具也在塑造我们。语言提升了人与人之间的信息交换效率，文字让知识可以跨代积累；今天这些小得多的软件当然不能和语言、文字相提并论，但它们也在给知识工作加上一些具体的“触角”。有的让信息更容易被找到，有的让资料更容易被理解，有的让想法更容易被留下，有的只是让电脑用起来少一点别扭。

所以这篇文章最想分享的不是某个软件的下载链接，而是一份需求清单。下一次你在某个重复动作上感到麻烦，或者只是隐约觉得“这件事怎么这么绕”，也许值得停一下，搜一搜：有没有人已经为这个小问题做过一件工具。
